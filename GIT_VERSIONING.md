# Инструкция по версионированию проекта на GitHub

## Вариант 1: Использование Git тегов (рекомендуется для релизов)

Это лучший вариант, если вы хотите сохранить старую версию и отметить новую как релиз.

### Шаг 1: Закоммитить изменения

```bash
# Добавить все изменения
git add .

# Закоммитить с описанием
git commit -m "Обновление: улучшения конфигурации и документации"

# Отправить на GitHub
git push origin main
```

### Шаг 2: Создать тег для версии

```bash
# Создать аннотированный тег (рекомендуется)
git tag -a v1.0.0 -m "Версия 1.0.0: Стабильный релиз"

# Или легковесный тег
git tag v1.0.0
```

### Шаг 3: Отправить тег на GitHub

```bash
# Отправить один тег
git push origin v1.0.0

# Или отправить все теги
git push origin --tags
```

### Результат:
- На GitHub будут доступны все версии по тегам
- Можно создать Release на GitHub для каждого тега
- Старая версия останется доступной через её тег

## Вариант 2: Создание новой ветки для версии

Если вы хотите продолжать разработку в разных версиях параллельно.

### Создать новую ветку для новой версии

```bash
# Создать и переключиться на новую ветку
git checkout -b v2.0.0

# Закоммитить изменения
git add .
git commit -m "Версия 2.0.0: новые улучшения"

# Отправить новую ветку на GitHub
git push origin v2.0.0
```

### Результат:
- `main` (или `master`) - старая версия
- `v2.0.0` - новая версия
- Обе версии доступны на GitHub параллельно

## Вариант 3: Комбинированный подход (рекомендуется для проектов)

### Для релизов используйте теги:

```bash
# 1. Закоммитить изменения
git add .
git commit -m "Подготовка к релизу v1.0.0"

# 2. Создать тег
git tag -a v1.0.0 -m "Релиз v1.0.0"

# 3. Отправить на GitHub
git push origin main
git push origin v1.0.0
```

### Для разных веток разработки:

```bash
# Работа над новой функциональностью в отдельной ветке
git checkout -b feature/new-feature
# ... работа ...
git push origin feature/new-feature
```

---

## Просмотр версий

```bash
# Список всех тегов
git tag

# Список всех веток
git branch -a

# Переключиться на конкретную версию по тегу
git checkout v1.0.0

# Вернуться на main
git checkout main
```

---

## Создание Release на GitHub

После создания тега на GitHub:

1. Перейдите на страницу репозитория
2. Нажмите "Releases" → "Create a new release"
3. Выберите созданный тег (например, v1.0.0)
4. Добавьте описание релиза
5. Нажмите "Publish release"

---

## Рекомендации по версионированию

Используйте [Semantic Versioning](https://semver.org/):
- `v1.0.0` - Major релиз (breaking changes)
- `v1.1.0` - Minor релиз (новые функции, обратная совместимость)
- `v1.0.1` - Patch релиз (исправления багов)
